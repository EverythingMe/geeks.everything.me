HackAthing is Everything's periodical hackathon. This Thursday (15th) we're teaming up with W3C and some 50 participants will be knee deep in HTML5 mobile webapps accompanied by Front End mentors, content and design experts and lots of fuel in the fridge.

It’ll start off with two extremely relevant lectures by Doron Nir on Gaming and yours truly on Mobile webapps. Afterwards we’ll talk over project ideas, split into groups and dive into code.



In my talk about Mobile Webapps I’ll touch key issues with development environment and debugging. Still, it could be helpful to prep up beforehand. So here’s the gist:

Get access to your localhost from your mobile device

The best and easiest method is via a proxy client on your host computer and a simple (yet evasive) Wifi setting on your mobile device. Once you set it, any local url will be accessible without having to fiddle with your hosts file.
As a bonus, all mobile network traffic can be monitored via the proxy app.

For Windows I recommend Fiddler. For Mac and Linux I go for Charles but WireShark will do if you know how to operate it.

Unfortunately, not all device operating systems have a proxy setting available. One workaround would be to simply browse to the local machine’s ip address (which has to be on the same network as the device) and make sure the site you’re working on is the default site of the web server.

There are quite a few more methods like using localtunnel, creating a hotspot and using a simulator or emulator.

Debugging and logging

Neither Android nor iOS (as of version 6) expose a log viewer within the device. This means a desktop comp must be used.
On Android, using the terminal command “adb logcat” will get you going with simple terminal viewer. This requires installing the Android SDK and it’s platform tools. I’ll show how in my HTML5Fest talk.

On iOS you gotta have a Mac with Safari 6. Yeah, I know… still the toolset they provide is the most comprehensive in mobile web debugging and if you get your hands on it you’ll be a happy camper.

One alternative to these is Weiner and it’s derivatives Adobe Edge Inspect and trigger.io.

Advanced APIs

Mobile web development has its rewards. Taking advantage of the webkit’s reliable rendering engine and advanced features in combination with mobile device capabilities – well, it makes it all worth while.

We’ll explore some of the features like the awesome Web Audio API (here's an introduction slideshow I had recently), device motion, orientation and touch events, geolocation data retrieval, css animations and much more.
