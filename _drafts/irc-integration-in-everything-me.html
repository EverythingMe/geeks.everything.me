Hello everyone,

We've recently introduced a new service for Everything.me employees for better collaboration - an internal IRC server to communicate with each other.
The idea sat in our CTO's mind (Joey Simhon) for a while, until some of our employees (including Joey) went for a week in Germany to work on FirefoxOS (Mozilla work week).
Aside from regular chat, the IRC has also introduced a new way to be notified on certain events and make collaboration even better! During the work week, tickets and bug reports were posted to IRC, alongside with GIT commits to Mozilla's repositories.

I've been assigned with the task of implementation of something similar, as I'm an old IRC user (been using IRC since 1994 or so...).
The requirements were simple enough - IRC server, ZNC (IRC bouncer) and GitHub's Hubot (node.js based robot with IRC support) - with lots of cool scripts to enhance its functionality.

I started off by installing ratbox-ircd (EFNet's IRCd), ZNC and Hubot with IRC adapter. The ZNC's job is to keep a chat buffer while you're offline and let you login simultaneously with multiple clients (and use the same nickname).
Then, I've installed Hubot with the IRC adapter, and installed some community scripts.
The bigger task was to integrate Hubot with our systems:

  • Hudson - our build server
  • Redmine - our ticketing/bug reports system
  • Gitlab - our GIT repositories system

The cool thing about Hubot is it has great integration with HTTP - it's written with node.js after all. It's very simple to send messages to the chat room based on HTTP requests the bot received.
